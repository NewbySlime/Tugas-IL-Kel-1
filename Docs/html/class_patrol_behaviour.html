Detailed Description

Behaviour class for patrolling system. The system uses PathFollower as its autonomous movement and a set of actions that a patrolling object can do. The sets of action nodes contained in namespace PatrolActions. This class also trigger Game Over event when a Game Over trigger object is registered. When an object has been registered to the trigger, there is a delay before a Game Over actually trigger. For creating custom patrol action, use interface PatrolAction.

This uses Visual Scripting but seperate from sequencing system. For setting the patrolling data, use PatrolBehaviour.SetInitData(PatrolBehaviour.InitData).

This class uses following component(s);

  • PathFollower component for pathfinding and AI movement.
  • MovementController for controlling movement system used in the Game.

This class uses external component(s);

  • AlertTrigger for watching potential "enemy" (in this case, Player) object to be registered as a Game Over trigger object.
  • BaseProgressUI Progress bar for "alerted" progress.
  • SoundAlertReceiver receiver for "imaginary sound" transmitted by SoundAlertTransceiver. This object used for diverting patrol object's attantion.
