Detailed Description

Class for translating user's input to available interactions in the Game. Some UI interaction handlings are detached from this component (ex. GameHandler handles the pausing system and UI).

This class uses following component(s);

  • MovementController for controlling movement system used in this Game.
  • Unity's PlayerInput for handling input from player.
  • HealthComponent for handling health system.
  • InventoryData for storing items.
  • RecipeDiscoveryComponent as to handle discoveries of undiscovered recipes at the start of the Game.
  • OnDeadGameOverTrigger as a separate trigger for triggering Game Over event when the object is dead.

This class uses external component(s);

  • WeaponHandler for controlling weapon system that can be used by player.
  • InteractionHandler for giving interaction interface between player and the Game's world objects.
  • PickableObjectPickerHandler for handling item picking system.
  • Unity's Animator Component for handling animation used for this object.
  • AnimationTriggerFlagComponent for listening to trigger flag that comes from Unity's animation system.
  • MouseFollower (gets by finding an instance) to get the mouse position.
  • MultipleProgressBar (gets by stored UI in GameUIHandler) as the weapon cooldown progress. r This class uses autoload(s);
  • GameHandler for Game events and such.
  • PersistanceContext for handling save files.
  • GameUIHandler for getting UI used in the Game.
  • ItemDatabase for getting data about certain item.
  • InputFocusContext for asking focus of input used.
